# Мониторинг безопасности (автоматизированное реагирование на угрозы)

Выполнил Сердюк Д.О.

Итоговое домашнее задание по Python. Скрипт анализирует логи веб-сервера, ищет подозрительные IP и имитирует блокировку.

## 📁 Состав проекта

- `threat_monitor.py` - основной скрипт
- `access.log` - файл с логами (пример)
- `threat_report.csv` - отчет после анализа (создается автоматически)
- `top_ips.png` - график топ IP (создается автоматически)
- `vt_cache.json` - кэш с результатами проверок (создается автоматически)
- `README.md` - этот файл

## 🔧 Как запустить

1. Установить библиотеки: `pip install requests pandas matplotlib`
2. Убедиться, что в папке есть файл `access.log` (приложил пример)
3. Запустить скрипт: `python threat_monitor.py`

## 📊 Что делает программа

1. Читает логи из файла `access.log`
2. Собирает все IP-адреса из логов
3. Считает, сколько раз каждый IP обращался к серверу
4. Проверяет каждый IP через "VirusTotal" (на самом деле имитация, чтобы не тратить ключи и лимиты)
5. Если IP вредоносный (красный) или подозрительный (оранжевый) - выводит предупреждение в консоль и имитирует блокировку
6. Сохраняет отчет в CSV файл (можно открыть в Excel)
7. Рисует график с топ-10 самых активных IP

## 🎨 Что означают цвета на графике

- **Красный** - вредоносный IP (из списка "плохих")
- **Оранжевый** - подозрительный IP (например, часто стучится в админку)
- **Синий** - нормальный IP (чистый)

## 📝 Пример отчета (CSV файл)

После запуска в папке появится `threat_report.csv` с такими колонками: `ip`, `requests`, `percent`, `malicious`, `suspicious`

## 👨‍💻 Для чего это нужно

Программа помогает быстро проанализировать логи сервера и понять: кто слишком часто стучится к нам на сайт, какие IP могут быть связаны с атаками, на кого обратить внимание админу.

## ⚠️ Важно

- Это учебный проект, всё работает "понарошку"
- Вместо реальных запросов к VirusTotal используется имитация со списком заранее известных IP
- Если хочешь подключить настоящий API, нужно получить ключ на сайте VirusTotal и доработать код
